1. 文档概述

1.1 背景与目标

本 PRD 聚焦于公域内容激励工具中两个核心已上线场景：互动加粉（账号加粉）和内容互动。这两个场景分别解决商家在私域运营中的两大痛点：

  • 互动加粉：商家需要将公域平台的流量沉淀为自有账号的粉丝，但缺乏低成本、规模化的引导手段
  • 内容互动：商家的重点笔记/短视频需要基础互动量（点赞、评论、收藏）来突破平台冷启动阈值，但缺少激励真实用户参与互动的工具

业务目标：

  • 帮助商家以任务激励方式批量引导用户关注官方社媒账号，实现账号矩阵导流
  • 帮助商家为重点公域内容快速积累真实互动数据，提升内容曝光和推荐权重
  • 建立「任务发布 → 用户参与 → 内容审核 → 奖励发放」的完整闭环

1.2 系统架构概览

本系统包含两个独立的前端界面：

界面端说明
用户端 H5（微信小程序）C 端用户浏览任务、提交凭证、查看审核状态和奖励
商家后台（SaaS 管理端）B 端商家运营人员配置任务、审核内容、查看数据看板
后台导航结构：后台侧边栏仅保留「任务管理」和「内容审核」两个入口。原「数据看板」已下线，/backend/dashboard 路径统一重定向至 /backend/tasks，任务管理作为后台主工作台。互动营销入口中「内容激励」直接跳转至任务管理页。

2. 场景一：互动加粉（账号加粉）

2.1 场景定义

商家创建「账号加粉」任务，指定需要用户关注的官方社媒账号列表，用户按要求关注后上传截图凭证，审核通过即获得奖励。适用于拉新关注、账号矩阵导流、会员关注认证等场景。

2.2 商家后台功能

2.2.1 任务创建与配置

入口：任务管理页面 → 场景卡片「账号加粉」→ 点击「去创建」→ /backend/tasks/create?scene=follow

页面结构：任务创建页分为两个 Tab——「任务配置」和「账号管理」。

Tab 1：任务配置
字段类型必填说明
任务名称文本是用户端可见，如「官方账号关注任务」
任务说明多行文本否对用户的补充说明
活动时间日期范围是开始日期 ~ 结束日期
关注奖励模式单选是「全部关注后统一奖励」/「每个账号独立奖励」
每人参与上限数字是默认 1，同一用户可参与次数
关注账号列表动态列表是从「账号管理」中已维护的账号中选择，至少 1 个
规则说明多行文本否对用户的规则补充描述
场景攻略描述卡片 + 开关否以图文卡片形式展示加粉流程三步骤（查看账号并完成关注 → 上传账号关注截图 → 审核通过发放奖励），可开关控制是否展示。关闭后下方关注账号列表和规则说明置灰不可编辑，校验放宽，保存按钮可正常提交
奖品配置奖品弹窗是支持积分、现金红包、赠品、抽奖机会四种类型
审核后允许重新提交开关是默认开启，拒绝后用户可重新上传截图

关注账号列表配置（每个目标账号）：

字段类型说明
平台下拉选择小红书、抖音
账号名称文本需关注的官方账号名称
示例图片图片上传展示给用户的关注成功示例截图
引导文案文本针对该账号的额外引导说明

拖拽排序：支持通过拖拽调整关注账号的展示顺序。

Tab 2：账号管理

商家在此维护可被任务关联的官方账号库。

字段类型说明
平台下拉选择小红书、抖音
账号名称文本官方账号名称
头像图片图片上传账号头像
累计粉丝数数字仅展示用
任务引流关注数数字展示该账号通过任务获得的关注数，以黄色徽章样式展示

业务规则：

  • 已被任务关联的账号不可删除，需先解除关联；删除时若该账号已被关注账号列表引用，弹窗提示「该账号已被任务关联，暂不可删除。请先在关注账号列表中替换或移除后再删除。」
  • 至少保留 1 个账号
  • 支持新增和删除账号
  • 账号上限最多 20 个，达到上限后新增按钮禁用
奖品配置弹窗
奖品类型配置项说明
积分积分数量如 80 积分
现金红包红包类型（拼手气/固定金额）、个数、总金额需接入微信支付
赠品赠品名称实物/虚拟赠品
抽奖机会关联转盘活动、发放次数通过弹窗选择转盘活动（如新客抽奖转盘、种草助推转盘、节日福利转盘），展示已选活动名称和 ID

奖品公共配置：

字段说明
奖品名称用户端可见的奖品名称
奖品图片用户端展示的奖品图片
奖品描述奖品的详细描述
抽奖机会配置：选择抽奖活动时弹出活动选择弹窗（替代下拉选择），展示预置的转盘活动列表。选定后展示活动名称和 ID，支持「活动管理」入口跳转。抽奖机会以紫色渐变徽章样式展示（带闪电图标）。

2.2.2 任务列表管理

入口：/backend/tasks

场景筛选：任务列表默认按场景筛选，支持「内容互动」为默认选项。任务列表页顶部展示 4 个场景创建卡片（账号加粉、内容互动、内容种草、效果种草），每个卡片有独立图标和配色。整张卡片可点击跳转至对应场景的创建页面，其中「账号加粉」和「内容互动」可正常创建，「内容种草」和「效果种草」卡片文案显示「敬请期待」。

2.2.3 内容审核（加粉凭证审核）

入口：/backend/review

列说明
用户信息头像 + 用户名
所属任务任务名称 + 任务ID + 场景标签
任务图片任务封面图（单图展示）
提交时间用户提交时间
审核状态待审核/已通过/已拒绝
所属平台平台徽章
审核人审核操作人（仅已通过/已拒绝时展示），默认显示「系统审核」
操作查看内容（外链）、通过、拒绝

2.2.4 任务数据详情

入口：任务列表 → 操作「数据」→ /backend/tasks/:id/data

KPI 卡片（5 个，加粉场景额外展示「奖品发放数量」）：

指标说明计算方式
总参与人数提交并参与的去重用户数COUNT(DISTINCT user_id)
内容提交量提交的凭证总条数COUNT(submission_id)
审核通过率通过数 ÷ 已审核数已通过 / (已通过 + 已拒绝) × 100%
积分发放总量已发放积分总和SUM(reward_amount) WHERE reward_type = 'points'
奖品发放数量活动中累计发放成功的奖品数量COUNT(reward_id) WHERE status = '已到账'

2.3 用户端 H5 功能

2.3.2 任务详情页

路由：/tasks/:id

场景一（加粉任务）详情展示：

区域内容
任务状态切换器浮动切换器，仅展示「进行中」和「已关闭」两个选项

2.3.5 奖品页面

路由：/rewards | 页面标题：「我的奖品」

Tab 筛选栏：全部 / 星币 / 赠品 / 红包 / 优惠券

奖品卡片展示：封面图(74×74) + 奖品名称 + 状态徽章（已领取/待发放/已失效）+ 中奖时间 + 查看按钮

2.4 后端业务逻辑

加粉任务创建逻辑：
1. 商家填写任务配置信息
2. 校验：任务名称非空、时间合法、账号列表非空、奖品配置非空
3. 生成任务 ID，初始化状态
4. 保存任务记录

加粉凭证审核逻辑：
- 通过 → 创建奖励记录 → 待到账
- 拒绝 → 记录原因 → 关联奖励已失效 → 通知用户

注：加粉任务支持「已关闭」状态，关闭后用户无法参与，但待审核的奖励通过会正常发放。

3. 场景二：内容互动

3.1 场景定义

商家创建「内容互动」任务，指定一条公域平台内容（笔记/短视频），要求用户对该内容完成指定的互动动作（点赞、评论、收藏），用户完成后上传互动截图凭证，审核通过即获得奖励。

3.2 商家后台功能

3.2.1 任务创建与配置

入口：/backend/tasks/create?scene=engagement

字段类型必填说明
任务名称文本是如「重点内容互动助推」
互动平台下拉选择是小红书、抖音
内容链接模式单选是「粘贴链接」/「上传分享图」。抖音平台仅支持链接模式
内容链接URL条件链接模式时必填
内容分享图图片上传条件分享图模式时必填，支持预览/删除/放大
互动动作多选是点赞、评论、收藏，至少 1 个
评论关键词文本条件勾选评论时必填
奖品配置奖品弹窗是积分、现金红包、抽奖机会

3.3.1 H5 任务详情页（互动场景）

活动状态切换器（浮动）：「未开始」「进行中」「已结束」

内容查看切换器（浮动）：「内容链接」和「分享图」，控制 ?contentView= 参数

3.4 后端业务逻辑

内容互动任务创建逻辑：
1. 校验：任务名称、时间、平台、互动动作、评论关键词、内容链接/分享图
2. 生成任务 ID，初始化状态，保存记录

互动凭证审核逻辑（同加粉场景）：
自动审核：截图质量、标题匹配、互动状态可见、关键词匹配
人工审核：账号匹配、动作完成、评论内容

6. 非功能需求

6.1 性能要求

指标要求
凭证提交响应时间< 3 秒（P90）
自动审核响应时间< 10 秒（P90）
平台整体并发500 QPS

6.3 风控规则

规则说明
提交上限每用户每任务最多 N 条（默认 1）
全局日限同一用户每日 ≤ 10 条
重复提交同一内容链接不可重复提交
奖励前校验到账前检查内容是否仍可访问

7. 关键业务流程图

7.1 互动加粉完整流程

商家创建加粉任务
    ├── 配置关注目标账号列表
    ├── 配置奖励规则
    └── 发布任务
         ▼
用户在小程序看到任务
    ├── 查看需关注的账号列表和示例截图
    ├── 逐个前往平台关注账号
    ├── 逐个上传关注截图
    └── 提交关注凭证
         ▼
审核流程（AI 自动 + 人工）
    ├── 通过 → 创建奖励 → 待到账 → T+N 到账
    └── 拒绝 → 记录原因 → 通知用户 → 可重新提交

注：加粉任务支持「已关闭」状态，关闭后用户无法参与。

7.2 内容互动完整流程

商家创建互动任务
    ├── 指定目标内容（链接/分享图）
    ├── 指定互动动作
    └── 发布任务
         ▼
用户在小程序看到任务
    ├── 点击「去互动」前往平台
    ├── 完成互动动作
    ├── 上传互动截图
    └── 提交互动凭证
         ▼
审核流程（AI 自动 + 人工）
    ├── 通过 → 创建奖励 → 待到账 → T+N 到账
    └── 拒绝 → 记录原因 → 通知用户

8. 两个场景的功能对比

维度互动加粉内容互动

9. 验收标准

9.1 互动加粉

  • 商家可成功创建加粉任务，配置关注目标和奖励
  • 任务列表正确展示加粉任务，筛选和操作功能正常
  • 用户端正确展示加粉任务详情和关注目标列表
  • 用户可为每个目标账号上传截图并提交凭证
  • 审核工作台正确展示加粉凭证，通过/拒绝操作正常
  • 审核通过后奖励正确进入待到账状态
  • 数据详情页 KPI 和明细数据正确

9.2 内容互动

  • 商家可成功创建互动任务，配置目标内容和互动动作
  • 用户端正确展示互动任务详情和目标内容
  • 用户可上传互动截图并提交凭证
  • 审核工作台正确展示互动凭证
  • 审核通过后奖励正确进入待到账状态

9.3 通用

  • 账号认证流程正常
  • 消息通知系统正常
  • 奖励引擎各类型奖励发放正确
  • 风控规则有效

10. 待确认项目

  • 第三方数据服务商选型及 API 能力边界
  • 商家积分/优惠券系统对接接口规范
  • 奖励到账时间策略（T+N 的 N 值是否可由商家自行配置）
  • AI 自动审核的付费模式和定价策略
  • 微信小程序订阅消息模板配置
  • 现金红包发放的微信企业付款资质审批进度
