# 互动加粉 & 内容互动 产品需求文档（PRD）

**版本**：v1.1
**日期**：2026-05-12
**状态**：更新稿

---

## 1. 文档概述

### 1.1 背景与目标

本 PRD 聚焦于公域内容激励工具中两个核心已上线场景：**互动加粉（账号加粉）** 和 **内容互动**。这两个场景分别解决商家在私域运营中的两大痛点：

- **互动加粉**：商家需要将公域平台的流量沉淀为自有账号的粉丝，但缺乏低成本、规模化的引导手段
- **内容互动**：商家的重点笔记/短视频需要基础互动量（点赞、评论、收藏）来突破平台冷启动阈值，但缺少激励真实用户参与互动的工具

**业务目标：**
- 帮助商家以任务激励方式批量引导用户关注官方社媒账号，实现账号矩阵导流
- 帮助商家为重点公域内容快速积累真实互动数据，提升内容曝光和推荐权重
- 建立「任务发布 → 用户参与 → 内容审核 → 奖励发放」的完整闭环

### 1.2 系统架构概览

本系统包含两个独立的前端界面：

| 界面 | 端 | 说明 |
|------|-----|------|
| 用户端 H5（微信小程序） | C 端 | 用户浏览任务、提交凭证、查看审核状态和奖励 |
| 商家后台（SaaS 管理端） | B 端 | 商家运营人员配置任务、审核内容、查看数据看板 |

**后台导航结构**：后台侧边栏仅保留「任务管理」和「内容审核」两个入口。原「数据看板」已下线，`/backend/dashboard` 路径统一重定向至 `/backend/tasks`，任务管理作为后台主工作台。互动营销入口中「内容激励」直接跳转至任务管理页。

## 2. 场景一：互动加粉（账号加粉）

### 2.1 场景定义

商家创建「账号加粉」任务，指定需要用户关注的官方社媒账号列表，用户按要求关注后上传截图凭证，审核通过即获得奖励。适用于拉新关注、账号矩阵导流、会员关注认证等场景。

### 2.2 商家后台功能

#### 2.2.1 任务创建与配置

**入口**：任务管理页面 → 场景卡片「账号加粉」→ 点击「去创建」→ `/backend/tasks/create?scene=follow`

**页面结构**：任务创建页分为两个 Tab——「任务配置」和「账号管理」。

##### Tab 1：任务配置

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务名称 | 文本 | 是 | 用户端可见，如「官方账号关注任务」 |
| 任务说明 | 多行文本 | 否 | 对用户的补充说明 |
| 活动时间 | 日期范围 | 是 | 开始日期 ~ 结束日期 |
| 关注奖励模式 | 单选 | 是 | 「全部关注后统一奖励」/「每个账号独立奖励」 |
| 每人参与上限 | 数字 | 是 | 默认 1，同一用户可参与次数 |
| 关注账号列表 | 动态列表 | 是 | 从「账号管理」中已维护的账号中选择，至少 1 个 |
| 规则说明 | 多行文本 | 否 | 对用户的规则补充描述 |
| 场景攻略 | 描述卡片 + 开关 | 否 | 以图文卡片形式展示加粉流程三步骤（查看账号并完成关注 → 上传账号关注截图 → 审核通过发放奖励），可开关控制是否展示。关闭后下方关注账号列表和规则说明置灰不可编辑，校验放宽，保存按钮可正常提交 |
| 奖品配置 | 奖品弹窗 | 是 | 支持积分、现金红包、赠品、抽奖机会四种类型 |
| 审核后允许重新提交 | 开关 | 是 | 默认开启，拒绝后用户可重新上传截图 |

**关注账号列表配置**（每个目标账号）：

| 字段 | 类型 | 说明 |
|------|------|------|
| 平台 | 下拉选择 | 小红书、抖音 |
| 账号名称 | 文本 | 需关注的官方账号名称 |
| 示例图片 | 图片上传 | 展示给用户的关注成功示例截图 |
| 引导文案 | 文本 | 针对该账号的额外引导说明 |

**拖拽排序**：支持通过拖拽调整关注账号的展示顺序。

##### Tab 2：账号管理

商家在此维护可被任务关联的官方账号库。

| 字段 | 类型 | 说明 |
|------|------|------|
| 平台 | 下拉选择 | 小红书、抖音 |
| 账号名称 | 文本 | 官方账号名称 |
| 头像图片 | 图片上传 | 账号头像 |
| 累计粉丝数 | 数字 | 仅展示用 |
| 任务引流关注数 | 数字 | 展示该账号通过任务获得的关注数，以黄色徽章样式展示 |

**业务规则**：
- 已被任务关联的账号不可删除，需先解除关联；删除时若该账号已被关注账号列表引用，弹窗提示「该账号已被任务关联，暂不可删除。请先在关注账号列表中替换或移除后再删除。」
- 至少保留 1 个账号
- 支持新增和删除账号
- 账号上限最多 20 个，达到上限后新增按钮禁用

##### 奖品配置弹窗

商家可为任务配置以下类型的奖品：

| 奖品类型 | 配置项 | 说明 |
|----------|--------|------|
| 积分 | 积分数量 | 如 80 积分 |
| 现金红包 | 红包类型（拼手气/固定金额）、个数、总金额 | 需接入微信支付 |
| 赠品 | 赠品名称 | 实物/虚拟赠品 |
| 抽奖机会 | 关联转盘活动、发放次数 | 通过弹窗选择转盘活动（如新客抽奖转盘、种草助推转盘、节日福利转盘），展示已选活动名称和 ID |

**奖品公共配置**：

| 字段 | 说明 |
|------|------|
| 奖品名称 | 用户端可见的奖品名称 |
| 奖品图片 | 用户端展示的奖品图片 |
| 奖品描述 | 奖品的详细描述 |

**抽奖机会配置**：
- 选择抽奖活动时弹出活动选择弹窗（替代下拉选择），展示预置的转盘活动列表（新客抽奖转盘、种草助推转盘、节日福利转盘）
- 选定后展示活动名称和 ID，支持「活动管理」入口跳转
- 抽奖机会以紫色渐变徽章样式展示（带闪电图标）

#### 2.2.2 任务列表管理

**入口**：`/backend/tasks`

**场景筛选**：任务列表默认按场景筛选，支持「内容互动」为默认选项。任务列表页顶部展示 4 个场景创建卡片（账号加粉、内容互动、内容种草、效果种草），每个卡片有独立图标和配色。整张卡片可点击跳转至对应场景的创建页面，其中「账号加粉」和「内容互动」可正常创建，「内容种草」和「效果种草」卡片文案显示「敬请期待」。

**列表字段**：

| 列 | 说明 |
|----|------|
| 任务名称 | 任务名称 + 任务ID |
| 任务场景 | 彩色标签展示场景类型 |
| 参与人数 | 当前参与的去重用户数 |
| 任务状态 | 未开始/进行中/已失效/已结束 |
| 任务时间 | 开始日期 → 结束日期 |
| 操作 | 编辑、数据、推广、复制、失效、删除 |

**操作说明**：

| 操作 | 说明 |
|------|------|
| 编辑 | 跳转至任务编辑页，修改任务配置 |
| 数据 | 跳转至任务数据详情页 |
| 推广 | 生成推广链接/二维码（预留） |
| 复制 | 复制任务为新草稿，名称追加「（副本）」 |
| 失效 | 将进行中的任务设为失效，需二次确认 |
| 删除 | 删除任务，不可恢复，需二次确认 |

**搜索与筛选**：
- 按任务名称搜索
- 按任务场景筛选（下拉）
- 按任务状态筛选（全部/未开始/进行中/已失效/已结束）
- 按所属平台筛选（下拉单选：全部/小红书/抖音）
- 支持重置筛选条件

**分页**：每页 10 条，支持翻页。

#### 2.2.3 内容审核（加粉凭证审核）

**入口**：`/backend/review`

审核工作台统一处理所有场景的提交内容，加粉场景的提交为用户上传的关注截图凭证。

**审核列表字段**：

| 列 | 说明 |
|----|------|
| 用户信息 | 头像 + 用户名 |
| 所属任务 | 任务名称 + 任务ID + 场景标签 |
| 任务图片 | 任务封面图（单图展示） |
| 提交时间 | 用户提交时间 |
| 审核状态 | 待审核/已通过/已拒绝 |
| 所属平台 | 平台徽章 |
| 审核人 | 审核操作人（仅已通过/已拒绝时展示），默认显示「系统审核」 |
| 操作 | 查看内容（外链）、通过、拒绝 |

**筛选条件**：
- 审核状态 Tab：待审核/已通过/已拒绝，各 Tab 展示数量统计
- 所属任务 ID
- 用户手机号
- 任务场景（账号加粉/内容互动/内容种草/效果种草）
- 所属平台
- 发布时间范围

**审核操作**：

- **通过**：弹窗二次确认，确认后内容状态变更为「已通过」，自动生成待发放奖励记录，发送消息通知用户
- **拒绝**：弹窗填写拒绝原因（必填），拒绝原因展示给用户，内容状态变更为「已拒绝」，关联奖励状态变更为「已失效」，发送消息通知用户

**AI 自动审核**（付费增值服务）：
- 功能弹窗展示 AI 自动审核的功能价值
- 开启后系统自动对新提交内容执行审核
- 审核结果同步到内容审核列表
- 基于内容标题、正文合规性、任务要求匹配度与风险特征进行自动判定

**分页**：每页 10 条。

#### 2.2.4 任务数据详情

**入口**：任务列表 → 操作「数据」→ `/backend/tasks/:id/data`

**KPI 卡片**（5 个，加粉场景额外展示「奖品发放数量」）：

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| 总参与人数 | 提交并参与的去重用户数 | COUNT(DISTINCT user_id) |
| 内容提交量 | 提交的凭证总条数 | COUNT(submission_id) |
| 审核通过率 | 通过数 ÷ 已审核数 | 已通过 / (已通过 + 已拒绝) × 100% |
| 积分发放总量 | 已发放积分总和 | SUM(reward_amount) WHERE reward_type = 'points' |
| 奖品发放数量 | 活动中累计发放成功的奖品数量 | COUNT(reward_id) WHERE status = '已到账' |

**数据明细**（两个子 Tab）：

**Tab 1：参与人明细**

| 列 | 说明 |
|----|------|
| 参与用户 | 头像 + 用户名 |
| 内容标题 | 提交的凭证标题 |
| 链接 | 「查看内容」跳转至原始提交 |
| 提交时间 | 用户提交时间 |
| 发布时间 | 凭证对应的发布时间 |
| 审核状态 | 已通过/待审核/已拒绝 |

筛选：按手机号查询、按审核状态筛选。

**Tab 2：奖品发放明细**

| 列 | 说明 |
|----|------|
| 用户昵称 | 头像 + 用户名 |
| 奖品类型 | 积分/赠品/现金红包/抽奖机会 |
| 奖品信息 | 奖品名称 + 奖品图片 |
| 发放时间 | 实际发放时间 |
| 领取状态 | 已领取/未领取 |

筛选：按手机号查询、按领取状态筛选。

**分页**：每页 10 条。

### 2.4 后端业务逻辑

#### 2.4.1 加粉任务创建逻辑

```
1. 商家填写任务配置信息
2. 校验：
   - 任务名称不能为空
   - 活动时间范围合法（开始 <= 结束）
   - 关注账号列表不能为空（至少 1 个）
   - 每个目标账号必须有平台和账号名称
   - 奖品配置不能为空
3. 生成任务 ID
4. 任务状态初始化为「未开始」（开始时间 > 当前时间）或「进行中」
5. 保存任务记录
```

#### 2.4.2 加粉凭证提交逻辑

```
1. 用户提交加粉凭证
2. 校验：
   a. 任务存在且状态为「进行中」
   b. 用户在该任务的提交次数未达上限
   c. 每个目标账号都有对应的截图凭证
3. 创建提交记录，状态为「待审核」
4. 发送消息通知用户「内容已提交，等待审核」
5. 如开启 AI 自动审核，触发自动审核流程
```

#### 2.4.3 加粉凭证审核逻辑

```
1. 审核人员对待审核记录执行操作
2. 通过：
   a. 更新提交状态为「已通过」
   b. 根据奖励配置创建奖励记录
   c. 奖励状态初始化为「待到账」
   d. 记录审核人和审核时间
   e. 发送消息通知用户「内容审核通过，奖励待到账」
3. 拒绝：
   a. 更新提交状态为「已拒绝」
   b. 记录拒绝原因
   c. 关联的奖励记录状态变更为「已失效」
   d. 发送消息通知用户「内容审核未通过」+ 拒绝原因
```

#### 2.4.4 奖励发放逻辑

```
1. 奖励审核通过后进入「待到账」状态
2. 根据 rewardMode 决定发放时机：
   - after_review：审核通过后 N 天发放（如 T+2）
   - after_end：活动结束后统一发放
3. 到账时间到达：
   a. 检查内容是否仍可访问（未删除、未设为私密）
   b. 如内容已删除，取消奖励（状态变更为「已失效」）
   c. 如内容正常，执行发放：
      - 积分类型：调用积分系统接口增加积分
      - 现金红包类型：调用微信支付接口发放红包
      - 抽奖机会类型：调用转盘活动接口增加抽奖次数
   d. 更新奖励状态为「已到账」
   e. 更新提交记录的奖励状态
   f. 发送消息通知用户「奖励已到账」
4. 发放接口需幂等性设计，防止重复发放
```

#### 2.4.5 数据统计逻辑

```
参与人数 = COUNT(DISTINCT user_id) WHERE task_id = {taskId}
提交量 = COUNT(submission_id) WHERE task_id = {taskId}
通过率 = COUNT(status='已通过') / COUNT(status IN ('已通过','已拒绝')) × 100%
积分发放总量 = SUM(amount) WHERE reward_type = 'points' AND status = '已到账'   -- 加粉场景
奖品发放数量 = COUNT(reward_id) WHERE task_id = {taskId} AND status = '已到账'  -- 内容互动场景
```

## 3. 场景二：内容互动

### 3.1 场景定义

商家创建「内容互动」任务，指定一条公域平台内容（笔记/短视频），要求用户对该内容完成指定的互动动作（点赞、评论、收藏），用户完成后上传互动截图凭证，审核通过即获得奖励。适合给重点笔记、短视频做互动助推。

### 3.4 后端业务逻辑

#### 3.4.1 内容互动任务创建逻辑

```
1. 商家填写任务配置信息
2. 校验：
   - 任务名称不能为空
   - 活动时间范围合法
   - 互动平台必须选择
   - 互动动作至少选择 1 个
   - 如选择「评论」，评论关键词不能为空
   - 内容链接/分享图至少填一项（根据模式）
   - 内容链接格式合法（URL 校验）
   - 奖品配置不能为空
3. 生成任务 ID
4. 任务状态初始化
5. 保存任务记录
```

#### 3.4.2 互动凭证提交逻辑

```
1. 用户提交互动凭证
2. 校验：
   a. 任务存在且状态为「进行中」
   b. 用户在该任务的提交次数未达上限
   c. 截图凭证已上传
3. 创建提交记录，状态为「待审核」
4. 发送消息通知用户
5. 如开启 AI 自动审核，触发自动审核流程
```

#### 3.4.3 互动凭证审核逻辑

审核流程与加粉场景一致。审核要点差异：

```
自动审核检查项：
1. 截图图片质量（非空白/模糊）
2. 截图中是否包含目标内容的关键信息（标题关键词匹配）
3. 截图中互动状态是否可见（点赞图标变色、收藏标记等）
4. 如有评论关键词要求，检查截图中评论内容是否包含关键词

人工审核检查项：
1. 截图中的账号信息是否与提交用户匹配
2. 互动动作是否全部完成
3. 评论内容是否符合要求
```

#### 3.4.4 奖励发放逻辑

与加粉场景一致，遵循相同的奖励类型、发放时机和到账机制。

**支持平台**：小红书、抖音、哔哩哔哩、微博

## 6. 非功能需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 凭证提交响应时间 | < 3 秒（P90） |
| 自动审核响应时间 | < 10 秒（P90） |
| 数据看板加载时间 | < 3 秒 |
| 单商家日提交量 | 支持 10,000 条 |
| 平台整体并发 | 500 QPS |

### 6.2 安全要求

- 用户账号绑定信息加密存储
- 所有 API 接口使用 JWT 鉴权
- 商家后台操作日志保留 180 天
- 奖励发放接口幂等性设计，防止重复发放
- 同一内容链接不可重复提交
- 同一公域账号不能被多个用户绑定

### 6.3 风控规则

| 规则 | 说明 |
|------|------|
| 提交上限 | 每用户每任务最多提交 N 条（商家配置，默认 1） |
| 全局日限 | 同一用户每日全局提交不超过 10 条（系统限制） |
| 重复提交 | 同一内容链接不可重复提交 |
| 账号唯一 | 同一公域账号不能被多个用户绑定 |
| 奖励前校验 | 奖励到账前检查内容是否仍可访问 |
| 内容删除处理 | 内容已删除时取消待到账奖励 |

### 6.4 兼容性

- 用户端：微信小程序（iOS + Android）
- 商家后台：Web 端，Chrome 90+、Edge 90+、Safari 14+

## 7. 关键业务流程图

### 7.1 互动加粉完整流程

```
商家创建加粉任务
    │
    ├── 配置关注目标账号列表
    ├── 配置奖励规则
    └── 发布任务
         │
         ▼
用户在小程序看到任务
    │
    ├── 查看需关注的账号列表和示例截图
    ├── 逐个前往平台关注账号
    ├── 逐个上传关注截图
    └── 提交关注凭证
         │
         ▼
系统接收提交
    │
    ├── 校验提交次数上限
    ├── 校验截图完整性
    └── 创建待审核记录
         │
         ▼
审核流程（AI 自动 + 人工）
    │
    ├── 通过 → 创建奖励记录 → 待到账 → T+N 到账 → 用户收到奖励
    └── 拒绝 → 记录原因 → 通知用户 → 用户可重新提交（如允许）
```

```
    └── 创建待审核记录
         │
         ▼
审核流程（AI 自动 + 人工）
    │
```

## 8. 两个场景的功能对比

| 维度 | 互动加粉 | 内容互动 |
|------|----------|----------|
| 场景目标 | 引导用户关注官方账号 | 引导用户对指定内容互动 |
| 用户操作 | 关注账号 + 上传截图 | 点赞/评论/收藏 + 上传截图 |
| 任务配置核心 | 关注目标账号列表 + 场景攻略开关 | 目标内容（链接/分享图）+ 互动动作 |
| 提交凭证 | 每个账号独立截图 | 1 张互动完成截图 |
| 审核重点 | 截图中「已关注」状态 | 截图中互动状态和评论内容 |
| 奖励模式 | 全部关注/逐个关注 | 单次完成即得 |
| 评论关键词 | 不适用 | 可选配置 |
| 内容链接 | 不适用 | 必须配置（链接或分享图） |
| 阶梯奖励 | 不适用 | 不适用（基础奖励） |
| 排名奖励 | 不适用 | 不适用 |
| 账号认证前置 | 不强制（截图验证） | 不强制（截图验证） |
| H5 状态切换 | 进行中/已关闭（2 选项） | 未开始/进行中/已结束（3 选项） |
| H5 Hero 横幕 | 不展示 | 展示 |
| 奖励进度 | 展示奖励进度条 | 不展示 |
| 内容查看模式 | 不适用 | 链接/分享图切换 |
