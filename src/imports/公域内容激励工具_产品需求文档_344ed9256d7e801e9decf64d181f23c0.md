# 公域内容激励工具 产品需求文档

**版本**：v1.0
**日期**：2026-04-16
**状态**：草稿

## 1. 文档概述

### 1.1 背景与目标

当前私域电商商家在用户运营上面临两大核心痛点：一是公域平台（小红书、抖音、哔哩哔哩等）的内容种草效果难以与私域转化形成闭环；二是激励用户主动创作、传播品牌内容（UGC）的工具缺失，导致品牌口碑增长依赖高成本 KOL 投放。

本功能旨在为商家提供一套**「公域种草 → 私域沉淀 → 闭环激励」**的 UGC 内容激励 SaaS 工具，帮助商家以低成本激励真实用户在公域平台发布内容、获取曝光，并通过奖励机制将公域流量反哺至私域小程序。

**业务目标：**

- 帮助商家降低内容获客成本，提升 UGC 内容产出量
- 建立公域内容与私域转化的数据闭环
- 沉淀可复用的品牌 UGC 素材资产

### 1.2 范围

**本期包含（MVP）：**

- 用户公域账号绑定与资质认证（小红书、抖音，哔哩哔哩）
- 任务发布与内容提交流程
- 内容链接审核（混合模式：自动 + 人工）
- 互动数据采集与阶梯奖励计算
- 积分 / 优惠券奖励发放
- 商家后台任务配置与数据看板

**本期不包含：**

- 哔哩哔哩、视频号平台支持（二期）
- 现金红包发放（需微信企业付款资质，三期）
- UGC 内容授权管理与素材库（二期）
- 防作弊机器学习模型（二期，本期用规则策略替代）
- 直播打卡、线下打卡场景扩展（后续迭代）

## 2. 用户与场景

### 2.1 目标用户

| 用户类型 | 特征描述 | 核心诉求 |
| --- | --- | --- |
| 商家运营人员 | 使用 SaaS 后台配置任务规则、审核内容、查看数据 | 低成本获取真实 UGC 内容，提升品牌公域曝光 |
| 商家终端用户（买家） | 在商家小程序内操作，绑定公域账号、提交内容链接 | 发布内容后获得积分、优惠券等实质奖励 |
| SaaS 平台运营 | 管理商家入驻、监控平台数据质量 | 保障平台内容合规，控制作弊风险 |

### 2.2 用户故事

- **作为** 商家运营，**我希望** 能自定义任务规则（指定话题标签、平台、奖励金额），**以便** 精准引导用户发布符合品牌需求的内容。
- **作为** 商家运营，**我希望** 系统能自动采集内容的点赞、评论数据并按规则发奖励，**以便** 减少人工核查工作量。
- **作为** 终端用户，**我希望** 能在小程序内一键绑定我的小红书账号，**以便** 快速参与任务并获得奖励。
- **作为** 终端用户，**我希望** 提交内容链接后能实时看到审核进度和奖励预估，**以便** 了解我的参与状态。
- **作为** 商家运营，**我希望** 查看内容激励任务的整体 ROI 数据，**以便** 评估活动效果并优化策略。

### 2.3 使用场景

**场景一：用户首次参与任务（账号绑定）**
用户在商家小程序首页看到「发内容赢积分」活动入口 → 点击进入任务中心 → 选择平台（小红书） → 系统生成专属验证码 → 用户将验证码填入小红书个人简介 → 点击「我已填写，验证」 → 系统抓取账号主页比对验证码 → 验证成功，账号绑定完成。

**场景二：提交内容链接获取奖励**
用户在小红书发布商家相关内容（含指定话题标签） → 复制笔记链接 → 在小程序任务中心点击「提交内容」 → 粘贴链接 → 系统自动解析（校验账号归属、关键词合规） → 进入审核队列 → 审核通过后奖励进入待到账状态 → 7 天后奖励自动发放至用户账户。

**场景三：商家配置激励任务**
商家运营登录 SaaS 后台 → 进入「内容激励」模块 → 新建任务 → 填写任务名称、指定平台、必须包含的话题标签/关键词 → 设置投稿奖励（发布即得 50 积分）和阶梯奖励（达 100 赞再得 100 积分）→ 设置每用户上限、活动时间 → 发布任务。

### 模块一：公域账号绑定与认证

**功能描述：**
用户在商家小程序内完成公域平台账号的所有权验证，系统记录账号与用户的绑定关系，后续内容提交时以此作为归属校验依据。

**交互逻辑（以小红书为例）：**

1. 用户进入「账号认证」页面，选择平台「小红书」
2. 系统为该用户生成唯一验证码（格式：`#品牌名+6位随机字符`，如 `#BRAND-A3X9K1`），有效期 30 分钟
3. 页面展示验证码及操作引导：「请将以上验证码复制到你的小红书个人简介中」
4. 用户操作完成后点击「我已填写，开始验证」
5. 系统调用数据服务接口，抓取用户输入的小红书主页 URL 对应账号的个人简介
6. 比对简介中是否包含验证码字符串
7. 验证成功：记录绑定关系（用户 ID + 平台 + 平台账号 ID + 账号主页 URL + 粉丝数快照），提示「验证成功，可删除简介中的验证码」
8. 验证失败：提示原因（超时/未找到验证码），允许重新生成验证码重试

**业务规则：**

- 每个用户每个平台只能绑定一个账号
- 同一个公域账号不能被多个用户绑定（全平台唯一性校验）
- 验证码有效期 30 分钟，超时需重新生成
- 绑定后用户仍可解绑，解绑后 24 小时内不能绑定新账号（防止频繁切换账号刷奖励）

**异常处理：**

- 网络异常导致抓取失败：提示「验证服务暂时不可用，请稍后重试」，不消耗重试次数
- 用户输入的主页 URL 格式错误：前端实时校验，给出格式提示
- 账号已被其他用户绑定：提示「该账号已被绑定，如有疑问请联系客服」

### 模块二：任务中心（用户侧）

**功能描述：**
用户在小程序内查看商家发布的内容激励任务，了解任务规则和奖励，提交内容链接并跟踪审核状态。

**交互逻辑：**

1. 用户进入任务中心，展示当前进行中的任务卡片列表（任务名称、奖励摘要、截止时间、已参与人数）
2. 点击任务卡片进入任务详情页：
    - 任务规则说明（需发布平台、必含话题标签、内容类型要求）
    - 奖励说明（投稿奖励 + 阶梯奖励规则）
    - 参与状态（未参与 / 审核中 / 已通过 / 未通过）
3. 点击「去发布」可跳转至对应平台 App（小程序外链）
4. 点击「提交内容」弹出提交面板：
    - 输入内容链接（支持小红书笔记链接、抖音视频链接）
    - 选择归属账号（如绑定了多个平台账号）
    - 提交后实时显示「解析中」状态，自动校验通过进入「审核中」
5. 「我的提交」列表展示历史提交记录及状态、奖励金额

**业务规则：**

- 用户需先完成对应平台账号绑定，才能提交该平台的内容
- 同一内容链接不可重复提交
- 每个用户每个任务最多提交 [商家配置，默认3条] 内容
- 每日全局提交上限：同一用户不超过 10 条（风控规则，不可见）

**异常处理：**

- 链接解析失败（内容已删除/私密）：提示「内容无法访问，请确认内容为公开状态」
- 账号归属不匹配：提示「该内容的发布者与你绑定的账号不一致」
- 超过提交上限：提示「本任务每人最多提交 X 条内容」

### 模块三：内容审核系统

**功能描述：**
系统对用户提交的内容链接进行合规性自动审核，无法自动判断的进入人工审核队列，商家运营人员在后台完成最终审核。

**自动审核逻辑：**

1. **链接合法性校验**：解析链接域名，确认为支持平台的合法内容链接
2. **账号归属校验**：通过数据服务接口获取内容发布者账号 ID，与用户绑定账号比对
3. **关键词/话题检测**：抓取内容标题、正文、话题标签，检测是否包含任务要求的关键词（精确匹配 + 模糊匹配）
4. **发布时间校验**：内容发布时间必须在任务有效期内
5. **内容类型校验**：任务如限定「仅图文」或「仅视频」则检查内容类型

**判定结果：**

- 全部规则通过 → 自动审核通过，进入奖励待发放队列
- 任意硬性规则不通过（账号不匹配、链接无效）→ 自动拒绝，通知用户
- 关键词检测模糊、内容质量判断等软性规则存疑 → 转人工审核队列

**人工审核界面（商家后台）：**

- 展示待审核内容列表（提交时间、用户昵称、内容预览缩略图、链接）
- 审核人员点击可查看完整内容（外部链接跳转）
- 操作：通过 / 拒绝（填写拒绝原因，用于通知用户）
- 支持批量通过

**业务规则：**

- 自动审核通过率目标 ≥ 70%（减少人工压力）
- 人工审核 SLA：提交后 48 小时内必须完成审核
- 审核结果通知：小程序内 push + 服务通知

### 模块四：奖励引擎

**功能描述：**
根据商家配置的奖励规则，在内容通过审核后计算应发奖励，并按设定时间发放至用户账户。

**奖励类型：**

| 奖励类型 | 触发时机 | 说明 |
| --- | --- | --- |
| 投稿奖励 | 内容审核通过后立即进入待到账 | 固定积分/优惠券 |
| 阶梯奖励-互动量 | T+3、T+7 数据快照时触发计算 | 按点赞/评论/收藏量分段给奖 |

**阶梯奖励规则示例（商家可配置）：**

- 点赞达 50：额外 +50 积分
- 点赞达 200：累计 +150 积分
- 点赞达 500：累计 +300 积分

**奖励发放流程：**

1. 内容审核通过 → 投稿奖励进入「待到账」（显示为 T+7 到账）
2. T+3 快照：采集当前互动数据，计算阶梯奖励增量，追加至待到账
3. T+7 快照：采集最终互动数据，完成最终计算
4. T+7 到账：将待到账奖励正式发放至用户账户，更新积分/优惠券余额

**业务规则：**

- 奖励到账前若发现内容已删除或账号违规，取消该内容的所有待到账奖励
- 已到账奖励不可撤回
- 每个用户单任务奖励上限由商家配置（防止单用户获取过高奖励）
- 积分有效期继承商家积分系统规则

**功能描述：**

| 内容类型限制 | 单选 | 不限/仅图文/仅视频 |
| 最低账号粉丝数 | 数字 | 0 为不限 |
| 任务有效期 | 日期范围 |  |
| 每用户最多提交数 | 数字 | 默认 3 |
| 投稿奖励类型 | 单选 | 积分/优惠券 |
| 投稿奖励金额/张数 | 数字 |  |
| 阶梯奖励规则 | 动态表单 | 最多配置 5 档 |
| 单用户奖励上限 | 数字 |  |
| 审核模式 | 单选 | 自动/人工/混合（推荐） |

**数据看板指标：**

- 任务参与人数、内容提交量、审核通过率
- 内容总估算曝光量（各平台点赞/播放量加总）
- 奖励发放总量（积分、优惠券数量）
- ROI 参考指标：每曝光成本（奖励总成本 / 总曝光量）

## 4. 非功能需求

### 4.1 性能要求

- 链接提交后，自动审核响应时间 < 10 秒（P90）
- 数据看板页面加载时间 < 3 秒
- 系统支持单商家日提交量 10,000 条，平台整体并发提交 500 QPS
- 第三方数据服务接口调用超时阈值：8 秒，超时降级为人工审核队列

### 4.2 安全要求

- 用户账号绑定信息加密存储，不明文保存第三方平台 token
- 所有 API 接口使用 JWT 鉴权
- 商家后台操作日志保留 180 天
- 奖励发放接口需幂等性设计，防止重复发放

### 4.3 兼容性

- 用户侧：微信小程序（iOS + Android）
- 商家后台：Web 端，支持 Chrome 90+、Edge 90+、Safari 14+
- 移动端浏览器访问商家后台需基本可用（响应式布局）

### 4.4 可用性

- 服务可用性 SLA：99.5%
- 第三方数据服务不可用时，系统降级为全人工审核模式，不影响提交流程
- 关键操作（奖励发放）异常时触发告警，30 分钟内人工介入

## 5. 数据需求

### 5.1 核心数据实体

| 实体 | 关键字段 | 说明 |
| --- | --- | --- |
| 用户平台账号绑定 | user_id, platform, platform_account_id, profile_url, follower_count, bound_at | 记录绑定关系 |
| 激励任务 | task_id, merchant_id, platform, keyword_rules, reward_config, status, start_at, end_at | 商家配置的任务 |
| 内容提交记录 | submission_id, user_id, task_id, content_url, platform_content_id, review_status, submit_at | 用户提交的内容 |
| 内容互动快照 | submission_id, snapshot_day, like_count, comment_count, collect_count, play_count | T+1/3/7 数据 |
| 奖励记录 | reward_id, submission_id, user_id, reward_type, amount, status, released_at | 奖励发放明细 |

### 5.2 数据统计与埋点

- **账号绑定发起**：`event=account_bind_start`，参数：platform, user_id
- **账号绑定成功/失败**：`event=account_bind_result`，参数：platform, result, fail_reason
- **内容提交**：`event=content_submit`，参数：task_id, platform, user_id
- **审核结果**：`event=review_result`，参数：submission_id, result, review_type（auto/manual）
- **奖励到账**：`event=reward_released`，参数：reward_id, reward_type, amount

## 6. 设计与交互要求

### 6.1 设计原则

- **用户侧**：轻量、引导清晰，流程步骤不超过 3 步；奖励可见性强，激励感突出
- **商家后台**：企业级 SaaS 风格，数据优先，配置表单分步引导降低误操作
- 整体视觉风格与商家品牌调性保持一致（主题色可配置）

### 6.2 关键页面/流程

**用户侧关键流程：**

```
任务中心首页
  ├── 任务卡片列表
  │     └── 任务详情页
  │           ├── 去发布（外跳平台）
  │           └── 提交内容
  │                 └── 链接输入 → 解析校验 → 提交成功
  └── 我的提交（历史记录）
        └── 提交详情（状态 + 奖励明细）

账号认证流程（首次参与前强制引导）
  └── 选择平台 → 获取验证码 → 去填写简介 → 验证 → 完成
```

**商家后台关键页面：**

```
内容激励模块
  ├── 任务管理（列表 + 新建/编辑）
  ├── 内容审核台（待审核队列 + 操作）
  └── 数据看板（任务维度 + 整体汇总）
```

## 8. 风险与依赖

### 8.1 风险点

| 风险 | 影响 | 应对措施 |
| --- | --- | --- |
| 第三方数据服务（飞瓜/蝉妈妈）接口不稳定或限流 | 高：影响自动审核和数据采集 | 接入多家服务商兜底；降级为人工审核 |
| 平台爬取规则变更导致链接解析失效 | 高：自动审核瘫痪 | 由数据服务商维护，我方关注变更通知；保留人工审核通道 |
| 用户刷量作弊（购买互动数据） | 中：奖励成本虚增 | 奖励延迟到账；异常增速预警（二期引入）；单用户上限控制 |
| 商家配置规则过于严苛导致用户提交通过率低 | 中：用户体验差，激励失效 | 商家后台提供规则预览和通过率预估提示 |
| 用户隐私合规（收集第三方平台账号信息） | 中：法律风险 | 用户协议明确告知数据使用范围；最小化数据采集；不存储平台登录凭证 |

### 8.2 外部依赖

- **第三方内容数据服务**：飞瓜数据 / 蝉妈妈 / 新榜 API，用于链接解析和互动数据采集（核心依赖，需提前完成商务签约）
- **商家积分系统**：奖励发放需与各商家自有积分/优惠券系统对接，需提前梳理接口标准
- **微信小程序服务通知**：审核结果推送依赖微信订阅消息，商家需配置消息模板
- **设计团队**：用户侧小程序 UI 稿和商家后台设计稿
